# Storybook Findings

- ✅ Use MUI components directly in Storybook stories for UI/UX demos
- ✅ Place `.stories.tsx` files in the same folder as the component
- ✅ Use clear, stable story export names and titles for consistent URLs
- ✅ Reference Storybook stories by their title and export name in documentation
- ✅ Restart Storybook after adding new story files or folders
- ✅ Use the Simple Browser in VS Code to preview Storybook
- ⁉️ How to create consistent URL paths, for documentation
- ❌ Don’t use custom styles for MUI components unless required by design
- ❌ Don’t create new components if MUI already provides the needed UI
